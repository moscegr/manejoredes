Semana 1: Introducción a Cisco IOS y CLI

Duración: 8 horas | Temas: Modos de IOS, Contraseñas y Configuración Básica.

1. Del Entorno Gráfico a la Línea de Comandos

A diferencia de los módems caseros (como vimos en el Parcial 1), los Switches y Routers empresariales no tienen pantalla ni una página web con botones. Se configuran mediante una pantalla negra escribiendo comandos. A este sistema operativo se le conoce como Cisco IOS (Internetwork Operating System).

Modos de Operación principales:

  • Router> Modo EXEC de Usuario: Solo permite ver información básica. No se puede configurar nada.
  • Router# Modo EXEC Privilegiado: Da acceso a todos los comandos de monitoreo y guardado (comandos show). Se accede escribiendo enable.
  • Router(config)# Modo de Configuración Global: Aquí se aplican los cambios que afectan a todo el equipo (nombres, contraseñas). Se accede escribiendo configure terminal.

Contexto Real

Para acceder a esta consola negra por primera vez en la vida real, necesitas un Cable de Consola (Azul claro) que conecta el Router físico al puerto USB/Serial de tu Laptop, y usar un programa como PuTTY o TeraTerm.

COM3 - PuTTY

Press RETURN to get started.


Router> enable

Router# configure terminal

Enter configuration commands, one per line. End with CNTL/Z.

Router(config)# hostname SW_Sistemas

SW_Sistemas(config)# enable secret class

SW_Sistemas(config)# line console 0

SW_Sistemas(config-line)# password cisco

SW_Sistemas(config-line)# login

SW_Sistemas(config-line)# exit

2. Configurando las Puertas de Enlace

Un Router conecta diferentes redes. Para hacerlo, cada puerto físico del router (llamado Interfaz) necesita tener una Dirección IP asignada. Esa IP se convertirá en la Puerta de Enlace Predeterminada (Default Gateway) de las computadoras conectadas a ese puerto.

Comandos de Configuración de Interfaz:

  • Entrar a la interfaz: interface gigabitEthernet 0/0
  • Asignar IP: ip address 192.168.10.1 255.255.255.0
  • Encender el puerto: no shutdown (Los puertos de los routers vienen apagados de fábrica por seguridad).

Comandos de Verificación (Troubleshooting):

  • Ping: ping 192.168.20.1 - Envía paquetes a un destino para ver si responde (prueba de conectividad básica).
  • Prueba Loopback: ping 127.0.0.1 - Comprueba que la tarjeta de red interna de nuestra propia computadora funciona correctamente.

3. Haciendo la red inteligente e invisible

Configurar IP por IP en 500 computadoras de una escuela sería imposible. Además, las IPs locales (192.168.x.x) no pueden navegar en internet porque no son válidas mundialmente. Para resolver esto, los routers tienen dos "superpoderes":

DHCP (Protocolo de Configuración Dinámica de Host)

El router crea una "alberca" (Pool) de direcciones IP. Cuando conectas tu celular a la red, el celular grita "¿Alguien me da una IP?", y el servicio DHCP del router le presta una automáticamente (IP, Máscara, Gateway y DNS) por un tiempo determinado (concesión).

NAT (Traducción de Direcciones de Red)

Las IP Privadas (ej. 192.168.1.50) no existen en internet. Cuando tu celular intenta abrir YouTube, el router intercepta el paquete, le quita tu IP privada, le pone la IP Pública de Telmex/Totalplay (Traducción/Enmascaramiento) y lo manda a internet. Cuando YouTube responde, el router deshace el truco y te lo entrega.

Ejercicio 1: Script de DHCP

Objetivo: Configurar el servidor DHCP en el bloc de notas.

  1. En tu archivo de texto, agrega la exclusión de IPs (para que el router no preste su propia IP):
    ip dhcp excluded-address 192.168.10.1
  2. Crea el pool de IPs:
    ip dhcp pool ALUMNOS
  3. Declara la red que vas a repartir:
    network 192.168.10.0 255.255.255.0
  4. Declara la puerta de enlace que tendrán los alumnos:
    default-router 192.168.10.1

  • Pide a un compañero de otra computadora que haga lo mismo y comparen la IP Pública. ¿Son iguales o diferentes? ¿Por qué?

Semana 4: Mantenimiento y Ruteo Default

Duración: 8 horas | Temas: Protocolo CDP, Ruteo Predeterminado y Respaldos TFTP.

Respaldo TFTP

Router IOS

copy run tftp

Server TFTP

Comandos Avanzados de Administración

Para administrar una red empresarial de forma profesional, un técnico debe saber cómo descubrir equipos vecinos, cómo enviar el tráfico desconocido a internet y cómo respaldar su trabajo en caso de que el router se queme.

  • Protocolo CDP (Cisco Discovery Protocol): Es un protocolo exclusivo de Cisco que sirve para "ver" qué otros equipos Cisco están conectados directamente a nuestros puertos, sin saber sus IPs. Comando: show cdp neighbors.
  • Ruteo Default (Ruta estática predeterminada): Es la salida de emergencia. Le dice al router: "Si alguien te pide una IP que no conoces, mándala por este puerto hacia el módem de internet". Comando: ip route 0.0.0.0 0.0.0.0 [Interfaz_Salida].
  • Respaldos TFTP: Si un router se descompone, volver a escribir todo el código toma horas. Usamos un Servidor TFTP para guardar una copia exacta de la configuración. Comando: copy running-config tftp.

  3. Escribe el comando: ip route 0.0.0.0 0.0.0.0 g0/1.
  4. Responde en tu cuaderno: ¿Por qué usamos puros ceros en ese comando?

Ejercicio 2: Respaldo en Simulador

  • En el Servidor, ve a la pestaña Services, entra a TFTP y asegúrate de que esté encendido (ON).
